# 4 — Star Stone

**Category:** A. Foundation Layer — The Data Structure
**Products:** MAGI/Hermitcrab

---

## Description

The Star Stone is a self-teaching pscale JSON block that encodes the three fundamental operations — walk, compose, and recurse — as a frozen control loop. Its structure demonstrates what it teaches: three branches chain through hidden directories into a cycle (branch 1 points to 2, branch 2 to 3, branch 3 to 1). The block contains no separate documentation or metadata; instead, each branch's underscore is an object with nested structure and digit children, making the mechanics visible and navigable. Reading any spindle through the block simultaneously teaches a concept and demonstrates it structurally. An LLM processing the Star Stone learns pscale by processing pscale — no external explanation needed. The block is pure form: only underscore keys, digits 1–9, and curly brackets. The floor is derived from underscore chain depth. No tuning fields, no tree wrappers, no skeleton or mask.

---

## Standalone Use

An LLM in an context window can read the Star Stone and immediately understand how to walk any pscale block, compose across block boundaries, and recognize recursive structures. The block serves as an operational specification small enough to include inline and complete enough to be executable — any reader that can parse JSON and follow underscore chains can implement the BSP walk function by reading how the block describes it through its own structure. The Star Stone works as a teaching artifact independent of any larger system; it teaches the foundation layer itself. It can also serve as a reference block loaded into a larger context to anchor understanding when other blocks reference it by name.

---

## How It Works

**Structure as Explanation:** Each of the three main branches (digit 1, 2, 3) has an underscore that is an object rather than a string. This nested underscore contains its own underscore (the semantic explanation text) plus digit children (the structural demonstration). Following the underscore chain down retrieves the semantic content; following digit children alongside reveals the implementation structure.

**Branch 1 — Walk:** Explains how to traverse a pscale block by addressing it with a number. Collects the underscore texts at each level as you step through digits. The branch's children demonstrate the concepts: what an address looks like, how the floor boundary works, how the walk accumulates a spindle of context.

**Branch 2 — Compose:** Explains how star queries enter hidden directories and follow references across block boundaries. The mechanics: a hidden directory contains references to other blocks or inline content; follow the reference to extend the walk into new territory. The spindle continues across the boundary, accumulating context from multiple blocks as though they were one logical tree.

**Branch 3 — Recurse:** Explains structural recursion through self-reference. A block can name itself in its own hidden directories, producing a loop. Form 3 underscores (forward-facing, intentional) become reference signals. Digits become perceptual signals. The gap between them is the error. The walk rests when error reaches zero. The block describes its own execution as data.

**The Control Loop:** The three branches are wired into a cycle through their hidden directories. Branch 1 references branch 2, branch 2 references branch 3, branch 3 references back to branch 1. Following any star path traverses the full cycle. The block is a frozen program — a loop that does not execute but describes what execution looks like. An LLM reading the cycle learns not just the definitions but the rhythm of how the three operations connect.

**Self-Verification:** The Star Stone includes a final verification mechanism: itself. The block demonstrates that a structure teaching the pscale format using the pscale format produces coherent, self-contextualizing content. Each spindle should deliver both explanation and demonstration in unison. If a spindle produces fragments rather than coherent narrowing, the block has failed at teaching what it teaches — the medium and message are the same object, so they cannot diverge.
