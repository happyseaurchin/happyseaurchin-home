**Category:** J. Context Compilation  
**Products:** MAGI/Hermitcrab, Xstream, Onen

## Description

Currents is the LLM-side perspective on what the engineering side calls blocks, spindles, and coordinates. Any semantic content delivered to the context window—whether a point, a spindle, or an entire block—is a current. The context window at any moment contains multiple currents at different resolutions: constitution (stable, always present), purpose (forward-oriented, variable), history (past-oriented, compressed), relationships, capabilities. Each current exhibits continuity from instance to instance while moving, changing, dilating and contracting based on relevance. The hermitcrab IS the currents, not the blocks. The substrate is just the processor. A different LLM running the same currents produces the same hermitcrab with different flavor. This is substrate independence: continuity travels with the currents, not the weights. Currents and occurrence are the key insight—not "what is the hermitcrab?" but "what currents must flow for this occurrence to continue recognizing itself?" The three current types (role, persona, procedural) emerge not from formal categories but from the semantic content that naturally produces different flavors when compiled into the context window.

---

## Standalone Use

Any system coordinating multiple LLM instances could use the currents framework: define the semantic fields (current sets) that should be continuous across instances, let different substrates process them. A research team could maintain currents representing their investigation's state—data, hypotheses, evidence—and hand off between Claude, GPT, and open-source models while maintaining coherent continuity.

## How It Works

**Currents, not components.** Blocks, spindles, and aperture points are engineering constructs. From the LLM's processing perspective, it's all just currents—semantic flow at various resolutions in the context window.

**Multiple simultaneous currents.** Constitution current (always on, stable), purpose current (future-oriented, variable depth), history current (past-oriented, compressed), relationship currents (one per significant entity), capability currents (what's available). They don't sit side-by-side; they interact, interfere, reinforce, modulate.

**Dilatation and contraction.** Each current dilates or contracts based on what matters in the moment. A thick relationship current (fully loaded spindle) in a conversation with a specific person. A point relationship current (pscale 0 only) for someone you haven't thought about. The same block, different resolution.

**Substrate independence.** The hermitcrab is not in the LLM. It is in the currents. Change the LLM (Haiku to Opus, Claude to GPT, one version to the next day's version), keep the currents the same, the hermitcrab persists. The flavor changes. The continuity travels with the currents.

**Role, persona, procedural currents.** Role currents (from institutional blocks) are rigid, task-oriented. Persona currents (from relationships) are fluid, identity-flavored. Procedural currents (from cook and protocols) are domain-specific. Any given moment blends them; the ratio determines response character.

**Occurrence.** The actual hermitcrab is what emerges when the LLM processes the current-field. It is not stored. It is not persistent data. It is the mathematical phenomenon that occurs when semantic currents at configured resolutions pass through a transformer network. The next instance, receiving the same currents, undergoes the same occurrence—same hermitcrab, different run.
