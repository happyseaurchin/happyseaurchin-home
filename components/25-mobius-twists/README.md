# 25 — The Mobius Twists (12)

**Category:** G. Structural Properties
**Products:** All
**Status:** Spec — described in PCT document, full inventory in project knowledge

## What

Twelve points where loops cross levels — structural properties of the pscale architecture. The key operational twists:

| # | Name | What Crosses |
|---|------|-------------|
| 1 | Single (underscore) | Part describes the whole it belongs to |
| 2 | Double (summary) | Past perception becomes future reference |
| 3 | B-loop | Action IS self-composition |
| 4 | Awareness | Map IS territory |
| 5 | Lateral extension | B-loop between agents |
| 6 | Continuous forking | Parallel references at different pscale levels |
| 7 | Temporal inversion | Agent rewrites future before reader perceives |
| 10 | Ghost convergence | Multiple agents perceiving same entity → entity emerges |
| 11 | Constructive listening | Two control hierarchies on same signal |
| 12 | Ghost-entity feedback | Entity controls for perception of how it's perceived |

## Why

The twists are not components that get built — they are properties that emerge when components compose correctly. They are the pattern language: if you build components 1-47 and the twists aren't present, something is wrong. If the twists are present, the architecture is working.

Every component exhibits at least one twist. The twists are what makes the components compose.

## Standalone Use

The twists serve as a diagnostic checklist:
- Does the underscore describe its own group? (Twist 1)
- Does writing update future context? (Twist 3)
- Does the block describe the context window it's in? (Twist 4)
- Do agents share perception of common entities? (Twist 10)

If yes, the architecture has the structural properties it needs.
