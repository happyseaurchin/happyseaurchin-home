# 31 — Institutional Block

**Category:** I. Relational Mechanics
**Products:** MAGI, Xstream, Onen
**Status:** Spec — described in psychosocial conformality document

## What

Role specifications producing rigid, predictable currents. Three current types:

| Type | Character | Example |
|------|-----------|---------|
| **Role** (institutional) | Bounded, predictable | Teacher, customer service, engineer |
| **Persona** (relational) | Fluid, contextual | Friend, collaborator, mentor |
| **Procedural** (cook/rules) | Pure task, no identity | Recipe execution, data processing |

The institutional/relational ratio determines engagement character. A high institutional ratio = formal, bounded, predictable. A high relational ratio = fluid, surprising, personal.

## Why

Agents need to operate in institutional contexts (customer service, education, healthcare) where predictability and role boundaries matter. The institutional block provides this without hard-coding — the role currents are pscale blocks that can be swapped, modified, and evolved.

In Onen: game rules are institutional frames. Character sheets are role currents. The tension between institutional (rules) and relational (character) IS the game.

## Standalone Use

Define institutional roles for any agent:
1. Write the role as a pscale block — what the role IS, what it does, what it doesn't do
2. Set the institutional/relational ratio in the concern structure
3. Higher institutional ratio = the agent stays in role more rigidly
4. Lower institutional ratio = more relational, personal engagement
